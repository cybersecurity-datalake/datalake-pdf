\section{Descrição do Dataset}

Esta seção apresenta a descrição estrutural do dataset utilizado neste trabalho, abordando sua modelagem como um grafo de conhecimento no Neo4j. São detalhadas as entidades que compõem o modelo, seus respectivos atributos, relacionamentos e a forma como os dados são organizados logicamente.

\subsection{Estrutura de Dados}
O dataset é modelado como um grafo de conhecimento composto por quatro tipos de entidades principais e seus respectivos relacionamentos. Essa estrutura permite representar vulnerabilidades de software e suas interações com fraquezas, pacotes afetados e referências externas.

\subsection{Formatos de Armazenamento}

O dataset é armazenado em um banco de dados orientado a grafos utilizando o Neo4j. Diferentemente de modelos relacionais ou colunar, o Neo4j não utiliza tabelas tradicionais, mas sim nós (nodes), relacionamentos (relationships) e propriedades (properties), permitindo uma representação natural das interações entre vulnerabilidades, fraquezas, pacotes e referências.

O armazenamento do grafo é persistente no formato nativo do Neo4j, baseado em arquivos binários internos do sistema de gerenciamento de banco de dados. Esse formato é otimizado para consultas por travessia de relacionamentos, reduzindo a necessidade de joins complexos.

Para fins de ingestão e interoperabilidade, os dados podem ser importados a partir de formatos estruturados intermediários, como CSV e JSON, que são amplamente suportados pelo Neo4j através de ferramentas como o \textit{LOAD CSV} e a \textit{Neo4j Import Tool}.
